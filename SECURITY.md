# Security Policy

VulnLab APK is intentionally vulnerable. Reports should focus on problems in
the repository, build process, documentation, or unintended behavior outside the
documented lab scope.

## Reporting

Report sensitive issues privately to `1200km@gmail.com`.

Do not open public issues for:

- accidental secrets
- unsafe release artifacts
- unintended remote behavior
- vulnerabilities in infrastructure used to host the project

## Scope

In scope:

- unintended vulnerabilities outside the documented lab design
- build or signing issues
- unsafe documentation that could affect non-lab devices
- repository supply-chain concerns

Out of scope:

- the documented intentional vulnerabilities
- exploitability of `com.vulnlab.insecureapp` inside an authorized lab
