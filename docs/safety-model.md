# Safety Model

VulnLab APK is intentionally vulnerable and must be handled like a lab target.

## Safe Use

- Install only on emulators or devices you own.
- Do not install on production phones.
- Do not use real accounts, contacts, photos, messages, or tokens.
- Reset or delete the emulator after lab work.
- Keep network access controlled when testing WebView, cleartext traffic, or
  dynamic code loading exercises.

## Repository Safety

The repository should contain source code, docs, screenshots, and build files.
It should not contain live malware, real credentials, private customer data, or
third-party target material.

## Vulnerability Boundaries

The app intentionally demonstrates vulnerable patterns. The lab goal is to teach
how to find, exploit, and fix these patterns in a controlled environment.
