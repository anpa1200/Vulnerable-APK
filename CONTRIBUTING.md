# Contributing

Contributions are welcome when they improve lab accuracy, reproducibility,
documentation, safety, or Android compatibility.

## Rules

- Do not add live malware.
- Do not add real credentials, tokens, domains, or third-party target data.
- Keep every vulnerability clearly intentional.
- Keep exploit commands scoped to local emulator or owned-device labs.
- Update docs when source behavior changes.
- Add tests or validation notes where practical.

## Good Contributions

- clearer vulnerability explanations
- Android version compatibility fixes
- build script improvements
- safer lab setup or teardown instructions
- screenshots that match current behavior
- mappings to OWASP Mobile Top 10, MASVS, or MASTG

## Pull Requests

Include:

- summary of changed vulnerability behavior
- affected source files
- test or emulator validation notes
- safety impact
